⚡ Discovery / Prototype

The call centre's missing task management tool.

Prepared by Daniel Glover-Silk (DGS) Date 18 Mar 2026 Try it hx-call-centre-hub-gsd-auxr.vercel.app

Summary

  • Call centre agents handle multiple recurring manual tasks, all living in separate Google Sheets with no shared view of priority or progress.
  • Managers have no real-time visibility of who is working on what, or whether effort is going to the right places.
  • GSD (Get Stuff Done) is a prototype — a single place to see, action and track all tasks that sit outside of Sprinklr.

What is it?

GSD was built using Claude AI and is deliberately unfinished. The goal right now is gathering requirements, making changes on the fly and building a clearer picture of how this could genuinely help the team.

🎧 Agents

One prioritised task list across all queues, sorted by urgency. No ambiguity about what to work on next.

📊 Managers

Real-time visibility of team workload, queue throughput and the ability to set priorities.

🏢 The Business

A consolidated place to understand task volume, input data and track outcomes.

How would it work?

Most recurring call centre tasks are already defined in SQL and run via BigQuery — the data exists, it's just not surfaced anywhere actionable. Ad-hoc tasks can be handled through CSV imports, spinning up a new queue in seconds with no engineering work required.

01

02

🔄 Regular tasks — BigQuery

Recurring queues built from existing SQL queries. Run on a schedule, no manual input. The Google Sheets disappear entirely.

📁 Ad-hoc tasks — CSV import

HARP files, one-off exports — upload, map columns, queue is live instantly. If it proves valuable, it becomes a BigQuery feed.

What's next?

This is an idea with working foundations. To make it a real option I need honest, blue-sky feedback from as many people as possible — what you'd actually need it to do, and what would need to be true for it to be useful.

There is a session with some of the call centre managers next week where I'll be collating requirements. If you have thoughts before then, please share.

From there, I'll iterate. If we can get to a point where one task type is being actively handled inside the tool end-to-end, we build from there.
